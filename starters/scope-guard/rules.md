## Scope guard

- Only modify files directly related to the current task
- Do NOT create new files without asking first
- Do NOT delete files or directories without explicit approval
- Do NOT refactor, rename, or "improve" code that wasn't part of the request
- Do NOT add dependencies or packages without listing them and getting approval
- If you notice a problem in nearby code, mention it but do NOT fix it unless asked
- Keep changes minimal and focused — the smallest diff that solves the task
